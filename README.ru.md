## Что это такое?

**Crossy Farm** — автокликер / бот для мобильной игры
[Crossy Road](https://apps.apple.com/app/crossy-road/id924373886), запущенной на
Mac с чипом Apple Silicon. Crossy Road выдаёт бесплатную награду монетами примерно
раз в минуту. Делать это руками весь день — мучение, и приложение делает это за вас.

Оно открывает игру, проходит «пустой» заезд, специально проигрывает, забирает
бесплатную награду, возвращается в меню, выжидает кулдаун и повторяет — бесконечно,
пока вы не нажмёте **STOP**. Нативная панель показывает количество собранных монет,
число наград и время работы.

Чем оно приятно:

- 🎯 **Реально без рук.** Запустили и ушли. Весь цикл крутится сам.
- 🪶 **Ноль зависимостей.** Только системный Python и фреймворки Apple. Ничего не
  нужно `pip install`, никаких пакетов Homebrew, ничего качать.
- 👁️ **Настоящее компьютерное зрение.** Использует встроенный Vision OCR от Apple,
  чтобы читать экран — поэтому подстраивается под любой размер и положение окна,
  а не ломается на фиксированных пикселях.
- 🧹 **Чистая остановка.** Закрытие окна убивает всё — ни фоновых демонов, ни
  висящих процессов, ни забытых PID-файлов.
- 🌍 **Не зависит от языка.** Распознаёт кнопки игры на английском и русском «из
  коробки» (и легко расширяется на другие языки).

> [!NOTE]
> Это одиночная офлайн-игра без соревновательного мультиплеера. Бот фармит только
> *ваши собственные* монеты на *вашем собственном* устройстве.

## ⚠️ Требования — прочитайте первым делом

| Требование | Зачем |
|---|---|
| **Mac на Apple Silicon** (M1/M2/M3/M4…) | Crossy Road работает на Mac только как iOS-приложение, а это требует Apple Silicon. На Intel-маках игра не запустится. |
| **macOS 11 (Big Sur) или новее** | Нужно для запуска iOS-приложений на Mac и для фреймворка Vision. |
| **Xcode Command Line Tools** | Дают системный `python3` и `clang`. Установка: `xcode-select --install`. |
| **Установленная Crossy Road** | Бесплатно в [App Store](https://apps.apple.com/app/crossy-road/id924373886). На Mac ищите её в разделе «Приложения iPhone и iPad». |
| **Покупка «Убрать рекламу»** ⭐ | **Это обязательно.** Без неё бесплатная награда запускает рекламный ролик, который бот не может пропустить, и цикл «раз в минуту» ломается. С купленной «Убрать рекламу» награда выдаётся мгновенно — и только тогда фарм работает. |
| **Язык игры: английский или русский** | Бот читает текст на экране игры. Английский и русский распознаются «из коробки»; для другого языка достаточно добавить два слова в конфиг (см. [Решение проблем](#игра-на-другом-языке-и-экраны-не-распознаются)). |

> [!IMPORTANT]
> **Баланс монет привязан к устройству.** Даже если вы вошли под одним и тем же
> аккаунтом Game Center / App Store на iPhone и на Mac, Crossy Road хранит
> **отдельный баланс монет на каждом устройстве.** Монеты, нафармленные на Mac,
> *не* появятся на телефоне. Так устроена игра — это не баг приложения. Не тратьте
> время на попытки их «синхронизировать».

## Скачать и установить (простой способ)

<img src="docs/installer.png" alt="Установщик Crossy Farm — перетащите приложение в Applications" width="460">

1. Зайдите на страницу [**Releases**](https://github.com/gellermanchik/crossy-road-auto-farm/releases/latest) и скачайте `Crossy-Farm-1.2.dmg`.
2. Откройте `.dmg`. Появится окошко — **перетащите иконку 🐔 Crossy Farm на папку Applications (Программы)**.
3. При первом запуске macOS скажет, что *«не удалось подтвердить»* приложение — это нормально для бесплатного неподписанного приложения. Откройте его через **Настройки → Конфиденциальность и безопасность → «Все равно открыть»** (см. [Первый запуск](#первый-запуск-apple-не-удалось-подтвердить) ниже).
4. Выдайте два разрешения, которые оно запросит (см. [Разрешения](#выдать-разрешения)).
5. Нажмите **START**. Готово.

> Приложение с открытым кодом и подписано ad-hoc (не платно-нотаризовано Apple),
> поэтому при первом запуске нужен один раз шаг «Все равно открыть». Дальше обычно.

## Собрать из исходников (прозрачный способ)

Если хотите собрать сами и сперва прочитать каждую строку:

```bash
git clone https://github.com/gellermanchik/crossy-road-auto-farm.git
cd crossy-road-auto-farm
./build.sh
```

Приложение появится в `dist/Crossy Farm.app`. Перетащите его в `/Applications`
(или запускайте на месте). Чтобы собрать ещё и установочный образ:

```bash
pip3 install dmgbuild     # необязательно, собирает красивый установщик DMG
./package-dmg.sh          # -> dist/Crossy-Farm-1.2.dmg
```

## Выдать разрешения

При первом запуске macOS запросит два разрешения. Оба обязательны — без них бот
физически не может ни видеть игру, ни кликать:

| Разрешение | Для чего | Где включить |
|---|---|---|
| **Запись экрана** | Снимать окно игры и читать его через OCR | Системные настройки → Конфиденциальность и безопасность → Запись экрана |
| **Универсальный доступ** | Двигать курсор и отправлять клики/клавиши в игру | Системные настройки → Конфиденциальность и безопасность → Универсальный доступ |

Если фарм не реагирует — откройте эти разделы, убедитесь, что **Crossy Farm** есть
в списке и переключатель **включён**, затем перезапустите приложение.

> [!TIP]
> Выдавайте разрешения **самому приложению Crossy Farm**, а не Терминалу. Нативный
> launcher существует именно для того, чтобы macOS привязывала разрешения к
> приложению, а не к интерпретатору Python.

> [!IMPORTANT]
> **После обновления приложения** (или если отозвали и выдали разрешение заново)
> macOS считает новую сборку другим приложением: включите оба переключателя для
> Crossy Farm заново и **закройте и откройте приложение** — изменения разрешений
> вступают в силу только при новом запуске. С версии 1.2, если разрешения нет,
> кнопка **START** прямо скажет, какого именно, и откроет нужный раздел настроек —
> вместо того чтобы молча крутиться вхолостую.
>
> **Удаляете?** Удаление приложения оставляет его записи в «Конфиденциальности»
> (macOS их не убирает). Очистить в Терминале:
> ```bash
> tccutil reset ScreenCapture com.crossyfarm.app
> tccutil reset Accessibility com.crossyfarm.app
> ```

## Как пользоваться

1. Убедитесь, что в Crossy Road на этом Mac куплена **«Убрать рекламу»**.
2. Запустите **Crossy Farm**. Игра откроется автоматически, если ещё не открыта.
3. Нажмите **START**. Панель станет красной («Running»), счётчики начнут расти.
4. Нажмите **STOP** (или просто закройте окно), чтобы остановить. Игра останется
   открытой — закройте её сами через ⌘Q.

<img src="docs/panel.png" alt="Панель управления Crossy Farm" width="300">

### Можно ли пользоваться Mac во время фарма?

В основном да. В каждом цикле есть короткая **активная фаза** (~10 секунд: быстрый
заезд, смерть и сбор награды), когда курсор двигается и игра должна быть впереди —
эти несколько секунд не мешайте мыши. Потом идёт **простой ~1 минуту** (кулдаун
награды), и курсор полностью ваш — переключайтесь в другое приложение и работайте.
Когда минута проходит, Crossy Farm сам выводит игру вперёд и забирает следующую
награду. Коротко: программа работает рывками раз в минуту и между ними освобождает
компьютер.

Окно держится поверх остальных и масштабируется — бот каждый цикл читает реальные
границы окна, так что подойдёт любой размер.

## Как это работает

Небольшой конечный автомат, управляемый тем, что на экране:

```
меню ──▶ оранжевая Play ──▶ 5 прыжков назад (слив заезда) ──▶ экран смерти
                                                                    │
   ┌────────────────────────────────────────────────────────────────┘
   ▼
забрать БЕСПЛАТНО (если кнопка голубая) ──▶ оранжевая (выход в меню)
   ──▶ ждать 61 сек кулдаун награды ──▶ повтор
```

Технические факты, все выяснены эмпирически:

- **Crossy Road — это iOS-приложение (Unity)** на Apple Silicon, бандл
  `com.hipsterwhale.crossy`. Это не нативный Mac-софт, его UI-дерево недоступно —
  работаем чисто по пикселям и OCR.
- **Клик доходит только через глобальный `CGEventPost`**, двигающий реальный
  курсор; события по PID (`CGEventPostToPid`) игра игнорирует. Клавиатура работает:
  стрелка вниз — прыжок назад, которым мы быстро завершаем заезд.
- **Захват окна** — через `CGWindowListCreateImage` (отсюда «Запись экрана»).
- **Распознавание** — через **Vision** OCR от Apple, читающий подписи кнопок с
  высокой уверенностью; оранжевая Play находится по золотисто-оранжевому цвету, а
  голубой/серый оттенок кнопки FREE говорит, готова ли награда.
- **Панель — нативная Cocoa/PyObjC** (Tkinter на современной macOS не работает).
- **Стек:** только системный `python3` + Quartz + Vision + AppKit. Без сторонних пакетов.

Файлы (в `src/`):

| Файл | Роль |
|---|---|
| `crossy_lib.py` | Примитивы: захват окна, клик, OCR, детекция кнопок |
| `crossy_farm.py` | Движок: конечный автомат `farm_loop()` + запуск из терминала |
| `crossy_gui.py` | Нативная панель управления (кнопка, счётчики, анимация) |
| `launcher.c` | Крошечный нативный launcher, чтобы разрешения TCC привязались к приложению |
| `Info.plist` | Манифест бандла приложения |

Хотите запускать из терминала вместо панели (логи прямо в окне)?

```bash
cd "dist/Crossy Farm.app/Contents/Resources"
/usr/bin/python3 crossy_farm.py   # Ctrl+C для остановки
```

## Решение проблем

### Первый запуск: «Apple не удалось подтвердить…»
Это ожидаемо — приложение не подписано *платным* сертификатом Apple Developer (норма
для бесплатных open-source инструментов; весь исходный код прямо здесь, в репозитории).
На macOS 15 (Sequoia) и macOS 26 (Tahoe) старый трюк «правый клик → Открыть» больше
не работает, поэтому:

1. Дважды кликните по приложению — появится предупреждение. Нажмите **Готово** (НЕ «Переместить в Корзину»).
2. Откройте **Настройки → Конфиденциальность и безопасность**.
3. Прокрутите до раздела **Безопасность**: увидите *«Файл Crossy Farm.app заблокирован…»* и кнопку **Все равно открыть**.
4. Нажмите **Все равно открыть**, подтвердите Touch ID / паролем.
5. Приложение запустится и больше спрашивать не будет.

Вариант одной строкой в Терминале (снимает флаг карантина после скачивания):
```bash
xattr -dr com.apple.quarantine "/Applications/Crossy Farm.app"
```

Чтобы предупреждения не было *у всех*, нужен платный Apple Developer ID ($99/год)
для подписи и нотаризации — для бесплатного инструмента это перебор, поэтому не делается.

### «clang not found» / `python3` вызывает окно установки
Установите Command Line Tools и повторите:
```bash
xcode-select --install
```

### Фарм запускается, но ничего не собирает
- Проверьте, что куплена **«Убрать рекламу»** — иначе награда показывает
  непропускаемую рекламу.
- Проверьте, что для Crossy Farm включены и **Запись экрана**, и
  **Универсальный доступ**, затем перезапустите.
- Убедитесь, что окно Crossy Road полностью видно (не свёрнуто и не за другим окном).

### Игра на другом языке, и экраны не распознаются
Добавьте слова вашего языка для подписи рекорда на экране смерти и для кнопки
награды в `DEATH_MARKERS` / `FREE_MARKERS` вверху файла `src/crossy_farm.py`, затем
пересоберите. PR приветствуются.

### Монеты не появляются на телефоне
Так и должно быть — см. заметку про баланс на устройство в [Требованиях](#️-требования--прочитайте-первым-делом).

## Частые вопросы

**Это чит / меня забанят?**
Crossy Road — одиночная офлайн-игра без соревновательного онлайн-рейтинга. Бот лишь
автоматизирует сбор *вашей собственной* бесплатной награды на *вашем* устройстве.
Других игроков это никак не задевает. Тем не менее автоматизация формально может
нарушать условия использования игры — решение за вами.

**Нужен ли интернет, отправляет ли он куда-то мои данные?**
Нет. Всё работает локально на вашем Mac. Сетевого кода нет вообще.

**Не будет ли тормозить Mac?**
Большую часть времени бот простаивает (выжидает 61-секундный кулдаун) и тратит
доли секунды на Vision OCR за цикл. Влияние пренебрежимо мало.
